# Workspace

## Overview

This project is a pnpm workspace monorepo using TypeScript, designed for managing VPN services. It provides a robust API server, a React-based VPN control panel, and advanced features for traffic obfuscation and cluster management. The system aims to offer a secure, high-performance, and user-friendly solution for VPN administration, with a focus on evading Deep Packet Inspection (DPI).

**Key Capabilities:**
- **VPN User and Profile Management:** Create, update, delete, block, and unblock VPN users and profiles.
- **Traffic Obfuscation (Anti-DPI):** Implement advanced techniques like TLS ClientHello Fragmentation, Multi-Transport Support, Automatic Transport Fallback, and uTLS Fingerprint Rotation.
- **Cluster Management:** Manage multiple VPN servers, monitor their status, and enable inter-node synchronization.
- **Routing Rule Management:** Define and manage routing rules for traffic, including presets.
- **Monitoring and Statistics:** Track traffic usage, server status, and auto-switch events.
- **Admin Panel:** A React-based UI with a cyberpunk theme for easy administration.

## System Architecture

The project is structured as a pnpm monorepo, enabling shared libraries and consistent tooling across packages. TypeScript is used throughout, with composite projects to manage dependencies and type-checking efficiently.

**Core Technologies:**
- **Monorepo Tool:** pnpm workspaces
- **Node.js:** v24
- **TypeScript:** v5.9
- **API Framework:** Express 5
- **Database:** PostgreSQL with Drizzle ORM
- **Validation:** Zod (`zod/v4`), `drizzle-zod`
- **API Codegen:** Orval (from OpenAPI spec)
- **Build Tool:** esbuild
- **Frontend Framework:** React + Vite
- **Routing (Frontend):** wouter v3
- **State Management (Frontend):** TanStack React Query

**Architectural Patterns & Design Decisions:**
- **Modular Monorepo:** Separates concerns into `artifacts` (deployable applications like `api-server`, `vpn-panel`) and `lib` (shared libraries).
- **Type-Safe API:** OpenAPI specification defines API contracts, used by Orval to generate type-safe React Query hooks for the frontend and Zod schemas for backend validation.
- **Database Abstraction:** Drizzle ORM provides a type-safe interface to PostgreSQL.
- **Anti-DPI Architecture:** Implements TLS ClientHello Fragmentation, Multi-Transport Support with Automatic Fallback, and uTLS Fingerprint Rotation. Dynamic generation of Xray JSON configurations incorporates these anti-DPI features.
- **Cluster Management:** Supports multiple `vpn_servers` and `cluster_nodes` with inter-node communication for synchronization secured by `CLUSTER_SECRET`. Synchronization changelog tracks entity changes.
- **UI/UX Design:** The `vpn-panel` features a dark cyberpunk aesthetic with teal neon accents, utilizing custom UI components (`CyberCard`, `CyberButton`, etc.) and Russian localization with thematic naming conventions.
- **Authentication:** All VPN panel routes are publicly accessible (no login required for the panel itself, but API calls require authentication).

**Feature Specifications:**
- **API Routes:** Comprehensive RESTful endpoints for user, profile, server, routing, monitoring, anti-DPI, and cluster management.
- **VPN Panel Routes:** `/` (dashboard), `/users`, `/profiles`, `/routing`, `/cluster`, `/settings`.
- **IP Detection:** Dashboard displays client's public IP using a dedicated API endpoint.
- **VLESS Defaults:** Generated VLESS URLs default to `happ.su` but are configurable.
- **Anti-DPI:** Configurable TLS ClientHello Fragmentation, Multi-Transport Support (TCP, WebSocket, gRPC, HTTP/2) with automatic fallback, uTLS Fingerprint Rotation, and Xray config generation.
- **Distributed Cluster Sync Engine:** Peer-to-peer cluster mesh with HMAC-SHA256 authentication for inter-node communication, background sync with last-write-wins conflict resolution, and full CRUD synchronization for users and profiles.

## External Dependencies

- **PostgreSQL:** Primary database for persistent storage.
- **Orval:** API codegen tool, used to generate type-safe API clients and validation schemas from OpenAPI specifications.
- **TanStack React Query:** Used in the frontend for data fetching, caching, and state management.
- **wouter:** Lightweight React router for frontend navigation.
- **Radix UI:** Used for building accessible UI components, specifically for tooltips in the `vpn-panel`.
- **Xray:** Underlying technology for VPN tunneling, with dynamic configuration generated by the system to incorporate anti-DPI features.
