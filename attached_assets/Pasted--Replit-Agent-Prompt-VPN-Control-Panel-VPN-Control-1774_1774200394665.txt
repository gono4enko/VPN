# Replit Agent Prompt: VPN Control Panel

## ЗАДАЧА

Создай полноценное веб-приложение **VPN Control Panel** — панель управления для личного VPN-сервера на базе Xray-core (VLESS+Reality).

Приложение развёрнуто на Replit и общается с офисным Linux-сервером через REST API.

---

## СТЕК

- **Backend**: Python + FastAPI + uvicorn
- **Frontend**: React (Vite) + TailwindCSS
- **БД**: SQLite (через SQLModel/SQLAlchemy) — хранит профили VPN, пользователей, статистику
- **Взаимодействие**: FastAPI backend ↔ Xray JSON config на сервере через SSH (paramiko) или через локальный API-агент

## ДЕТАЛЬНОЕ ОПИСАНИЕ ФУНКЦИЙ

### 1. DASHBOARD (главный экран)

- Статус Xray-сервера: работает / не работает (кнопка restart)
- Текущий активный исходящий профиль (какой иностранный VPN используется)
- Количество подключённых клиентов
- График трафика за последние 24 часа (входящий/исходящий, библиотека recharts или chart.js)
- Ping до текущего активного иностранного VPN (показывается в ms)
- Кнопка "Переключить на самый быстрый" — запускает speedtest всех профилей и переключает

---

### 2. УПРАВЛЕНИЕ ПОЛЬЗОВАТЕЛЯМИ (страница Users)

Каждый пользователь = UUID в Xray inbound clients.

**Таблица пользователей:**
- Имя / описание
- UUID (скрыт, кнопка "показать")
- Flow (xtls-rprx-vision)
- Лимит трафика (GB, 0 = без лимита)
- Потреблено трафика (из access.log Xray)
- Статус: активен / заблокирован
- Дата истечения (опционально)
- Действия: Редактировать, Заблокировать/Разблокировать, Удалить, Скопировать URL, QR-код

**Добавление пользователя:**
- Форма: имя, лимит трафика, срок действия
- UUID генерируется автоматически (uuid4)
- После создания — показать VLESS URL и QR-код для импорта в телефон

**VLESS URL формат для офисного сервера:**
```
vless://{uuid}@{OFFICE_IP}:443?encryption=none&flow=xtls-rprx-vision
  &security=reality&sni=apple.com&fp=random
  &pbk={OFFICE_PUBLIC_KEY}&sid={OFFICE_SHORT_ID}
  &type=tcp#ИМЯ_ПОЛЬЗОВАТЕЛЯ
```

**QR-код**: генерировать через библиотеку `qrcode` (Python) или `qrcode.react` (frontend)

---

### 3. УПРАВЛЕНИЕ ИНОСТРАННЫМИ VPN ПРОФИЛЯМИ (страница Profiles)

Профиль = данные исходящего VLESS/Shadowsocks/WireGuard сервера.

**Добавление профиля — 4 способа:**

**Способ А: Импорт по URL**
- Поле ввода VLESS URL: `vless://UUID@IP:PORT?params#name`
- Кнопка "Импортировать"
- Парсинг URL → заполнение всех полей автоматически

**Способ Б: Импорт по подписке (Subscription URL)**
- Поле URL подписки (base64-encoded список серверов)
- Загрузка → парсинг → показ списка серверов для выбора
- Импорт выбранных

**Способ В: Скан QR-кода**
- Кнопка "Сканировать QR" → открывает камеру (jsQR или html5-qrcode)
- Декодирует VLESS URL из QR

**Способ Г: Ручной ввод**
- Форма: Протокол (VLESS/Shadowsocks/WireGuard), Адрес, Порт, UUID/Password, Flow, Security, SNI, Public Key, Short ID, Fingerprint, Название, Страна (эмодзи флаг)

**Карточка профиля показывает:**
- Флаг страны + название
- Протокол + адрес:порт
- Пинг (ms) — кнопка обновить
- Статус: активный / неактивный / ошибка
- Кнопки: Активировать, Тест скорости, Редактировать, Удалить

**Автовыбор лучшего профиля:**
- Кнопка "Авто" на Dashboard запускает параллельный ping всех профилей
- Сортирует по пингу
- Применяет профиль с наименьшим RTT в Xray outbound
- Перезапускает Xray
- Показывает результат: "Переключено на 🇳🇱 Amsterdam (12ms)"

**Расписание автопереключения:**
- Включить/выключить автопереключение
- Интервал проверки: каждые N минут
- Порог переключения: переключать если текущий профиль хуже лучшего на X ms

---

### 4. ОГРАНИЧЕНИЕ ДОСТУПА

**На уровне пользователей:**
- Лимит трафика (GB): cron каждые 5 мин парсит access.log → обновляет БД → если превышен лимит, удаляет UUID из Xray config и перезапускает
- Дата истечения: при просрочке — автоблокировка
- Ручная блокировка: убирает UUID из inbound, не удаляя из БД

**Белый список IP (опционально):**
- Список разрешённых IP для подключения к офисному серверу
- Применяется через iptables правила (команды через SSH)

### 5. XRAY MANAGER (backend, xray_manager.py)

```python
class XrayManager:
    def __init__(self, config_path: str):
        self.config_path = config_path  # /usr/local/etc/xray/config.json
    
    def read_config(self) -> dict
    def write_config(self, config: dict) -> None
    def restart_xray(self) -> bool          # subprocess: systemctl restart xray
    def get_status(self) -> dict            # systemctl is-active xray
    def add_client(self, uuid: str, flow: str) -> None
    def remove_client(self, uuid: str) -> None
    def set_outbound(self, profile: VpnProfile) -> None   # меняет outbound на профиль
    def get_active_outbound(self) -> str
    def parse_access_log(self) -> dict      # трафик по UUID из /var/log/xray/access.log
    def ping_host(self, host: str, port: int) -> int      # RTT в ms через TCP connect
```

### 6. API ENDPOINTS (FastAPI)

```
POST   /api/auth/login              # JWT токен
POST   /api/auth/logout

GET    /api/server/status           # статус Xray, uptime, версия
POST   /api/server/restart          # перезапуск
GET    /api/server/config           # текущий конфиг (без приватных ключей)

GET    /api/users                   # список пользователей

PUT    /api/users/{id}              # обновить (лимиты, имя)
DELETE /api/users/{id}              # удалить
POST   /api/users/{id}/block        # заблокировать
POST   /api/users/{id}/unblock      # разблокировать
GET    /api/users/{id}/qr           # PNG QR-код

GET    /api/profiles                # список VPN профилей
POST   /api/profiles                # добавить вручную
POST   /api/profiles/import-url     # импорт из VLESS URL
POST   /api/profiles/import-sub     # импорт подписки
PUT    /api/profiles/{id}           # обновить
DELETE /api/profiles/{id}           # удалить
POST   /api/profiles/{id}/activate  # применить как outbound
GET    /api/profiles/{id}/ping      # пинг профиля

POST   /api/speedtest/run           # тест всех профилей параллельно
POST   /api/speedtest/auto-select   # выбрать и применить лучший
```

### 9. ДЕПЛОЙ НА REPLIT

**replit.nix или .replit:**
- Запускать: `uvicorn backend.main:app --host 0.0.0.0 --port 8000` + `npm run dev` для frontend
- Или собрать frontend в static files и раздавать через FastAPI `StaticFiles`

**Рекомендуется второй вариант** (single process):
1. `cd frontend && npm run build` → создаёт `frontend/dist/`
2. FastAPI монтирует `dist/` как StaticFiles на `/`
3. Один сервис, один порт

**Startup команда для Replit:**
```bash
cd frontend && npm install && npm run build && cd .. && pip install -r requirements.txt && uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

### 10. ВАЖНЫЕ ДЕТАЛИ РЕАЛИЗАЦИИ

1. **Парсинг VLESS URL**: `urllib.parse.urlparse` + `parse_qs` для query params
2. **Парсинг подписки**: base64 decode → split('\n') → каждая строка = vless:// URL
3. **Ping**: TCP connect timeout к host:port, измерить RTT — не ICMP, а именно TCP (работает через firewall)
4. **QR-код**: библиотека `qrcode` (pip) → PNG → base64 → Data URL для img src
5. **Изменение конфига Xray**: читать JSON → изменить нужный раздел → записать → `subprocess.run(["systemctl", "restart", "xray"])`
6. **Если Replit не имеет доступа к офисному серверу напрямую**: добавить SSH-клиент (paramiko) и выполнять команды удалённо — это настраивается через .env переменные (SSH_HOST, SSH_USER, SSH_KEY)
7. **Безопасность**: все API endpoints кроме /login требуют Bearer JWT токен
